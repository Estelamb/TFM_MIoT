\chapter{Web Interface Views}\label{ap:ui}

This appendix provides the visual presentation, screenshots, and detailed functional descriptions of the authentication view and the six main dashboard pages of the \gls{AURA} platform.

\section{Authentication / Login}

Allows operators to authenticate with their credentials. Upon successful login, the application retrieves a \gls{JWT} access token, which is stored in the browser's context and automatically attached to all subsequent \gls{API} requests.

\section{Overview}

Provides a fleet-level summary containing cards for active devices (online vs. offline), active training and compilation jobs, and deployment statuses. It also lists recent edge activity and state transitions.

\section{Devices}

Displays the list of registered edge devices along with their real-time connection status (online/offline) and metadata. Clicking on a device navigates to its detailed view, showing hardware specifications, active peripheral sensors/actuators/others list. It allows to register a device with its hardware architecture and the peripherals.

\begin{figure}[H]
    \centering
    %\includegraphics[width=0.9\linewidth]{images/screenshot_devices.png}
    \caption{Devices Catalog and Live Status Indicators}
    \label{fig:screenshot_devices}
\end{figure}

\section{Models and Datasets}

A consolidated registry for custom \gls{AI} models, YOLO models, and datasets. It allows operators to:
\begin{itemize}
    \item Upload and manage dataset ZIP packages.
    \item Upload PyTorch models (\texttt{.pt}).
    \item Configure and trigger model training/retraining jobs asynchronously by specifying the epochs, input size, and batch size.
\end{itemize}

    \centering

    \caption{Models, Datasets and Training/Retraining Trigger Interface}
    \label{fig:screenshot_models}
\end{figure}

\section{Scripts}

A terminal-styled editor designed to author custom post-process python scripts. It includes an \gls{AST} validator that statically checks the process function contract and imports whitelist before allowing the operator to upload the script to the MinIO registry.

\begin{figure}[H]
    \centering

    \caption{Script Editor and \acrshort{AST} Validation Output}
    \label{fig:screenshot_scripts}
\end{figure}

\section{Deployments}

Manages and tracks the delivery of compiled models and scripts to edge devices. When a model is not yet compiled for a target architecture, creating a deployment automatically invokes the compilation job on the \gls{MLOps} Service in the background before sending the \gls{OTA} payload coordinates to the edge device.

\section{Monitoring}

Renders real-time telemetry analytics charts, including \gls{CPU}/\gls{RAM} logs, system temperature readings, and inference statistics (predictions, latency, and model drift indicators) sent by edge devices.
