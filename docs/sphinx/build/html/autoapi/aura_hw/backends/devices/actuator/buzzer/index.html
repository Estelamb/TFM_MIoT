Bases: aura_hw.backends.devices.actuator.base.ActuatorBackend

General Buzzer Actuator library/wrapper. Dynamically loads custom buzzer actuator from hardware/actuators/buzzer/<driver>/library.py.

property device_type: str[source]¶

Device category string.

Returns:

One of "camera", "sensor", "gpio".

property driver: str[source]¶

Driver / backend implementation name.

Returns:

A short lowercase identifier such as "opencv", "libcamera", "imx500", "bme280".

open(params: dict) → None[source]¶

Initialise the device with the given configuration parameters.

Parameters:

params – Dict from the params block in components_config.yaml for this component.

Raises:

  • RuntimeError – If the required SDK/driver is not available.

  • OSError – If the device cannot be opened (e.g. wrong index).

close() → None[source]¶

Release all resources held by this device.

Safe to call even if open() has not been called (no-op).

write(value: Any) → None[source]¶

Set or trigger the actuator state/action.

Parameters:

value – The state/command value (e.g. RGB color tuple for LED, bool for relay/buzzer).

Raises:

  • RuntimeError – If the actuator is not open.

  • IOError – If writing to the actuator fails.

  • driver — same as driver
