.. py:function:: execute_inference(inputs: numpy.ndarray | dict | None = None) -> Any

   Run a single inference pass with the currently loaded model.

   :param inputs: Input tensor(s) for the model.

                  * For Hailo and TFLite: a ``numpy.ndarray`` (NCHW for YOLOv8)
                    or a ``dict`` mapping input names to arrays.
                  * For RPi AI Camera (IMX500): pass ``None`` — the sensor
                    captures and runs inference internally.

   :returns:

             * Hailo: ``dict[str, numpy.ndarray]``
             * TFLite: ``dict[str, numpy.ndarray]``
             * IMX500: output from ``IMX500.get_outputs(metadata)``
   :rtype: Raw model outputs. Structure depends on the backend

   :raises RuntimeError: If :func:`load_model` has not been called yet.

   .. rubric:: Example

   >>> import numpy as np
   >>> tensor = np.zeros((1, 3, 640, 640), dtype=np.float32)
   >>> outputs = execute_inference(tensor)


.. py:function:: get_hardware_info() -> dict

   Return a snapshot of the current hardware and model state.

   :returns:

             * ``hardware_type`` (str): Detected hardware identifier.
             * ``model_loaded`` (bool): Whether a model is currently loaded.
             * ``backend`` (str | None): Class name of the active backend,
               or ``None`` if no model is loaded.
             * ``device_info`` (dict): Accelerator-specific metadata from
               :meth:`~aura_hw.backends.inference.base.InferenceBackend.device_info`.
   :rtype: A dict with the following keys

   .. rubric:: Example

   >>> info = get_hardware_info()
   >>> print(info)
   {'hardware_type': 'hailo8', 'model_loaded': True,
    'backend': 'HailoBackend', 'device_info': {...}}


.. py:function:: get_last_inference() -> Any

   Return the result of the most recent :func:`execute_inference` call.

   Allows the telemetry loop to read the latest inference result without
   triggering a new inference pass.

   :returns: The last inference output, or ``None`` if no inference has been
             run yet in this session.


.. py:function:: load_model(model_path: str) -> None

   Detect hardware and load a compiled model.

   Calls :func:`~aura_hw.detect.detect_hardware`, selects the matching
   backend, and invokes its ``load()`` method.  Any previously loaded
   model is **not** automatically unloaded — call :func:`unload_model`
   first if you need to swap models.

   :param model_path: Absolute path to the compiled model file
                      (``.hef`` for Hailo, ``packerOut.zip`` for IMX500,
                      ``.tflite`` for CPU).

   :raises RuntimeError: If the detected hardware has no supported backend.

   .. rubric:: Example

   >>> from aura_hw import load_model, execute_inference
   >>> load_model("/tmp/aura/model")


.. py:function:: unload_model() -> None

   Unload the current model and release accelerator resources.

   Safe to call even if no model is loaded (no-op in that case).


.. py:class:: DeviceManager(config_path: pathlib.Path)

   Lifecycle manager for all connected device backends.

   Reads ``components_config.yaml``, instantiates one backend per
   enabled component, and exposes a keyed-access API.

   Parameters
   ----------
   config_path:
       Absolute path to ``components_config.yaml``.


   .. py:method:: open_all() -> None

      Open all enabled device backends.

      Skips components whose drivers are not registered or that fail
      to open (logs an error but does not raise).



   .. py:method:: close_all() -> None

      Close all open device backends in reverse instantiation order.



   .. py:method:: get_device(component_id: str) -> aura_hw.backends.devices.base.DeviceBackend

      Return the backend for a specific component.

      :param component_id: The ``id`` field from ``components_config.yaml``.

      :returns: The instantiated :class:`~aura_hw.backends.devices.base.DeviceBackend`.

      :raises KeyError: If no enabled component with that ID exists.



   .. py:method:: get_all_info() -> dict[str, dict]

      Return ``info()`` for every managed device.

      :returns: Dict mapping ``component_id → info_dict``.



   .. py:method:: list_components() -> list[str]

      Return the IDs of all managed (enabled) components.
